# Windows 10/11 发布验收

自动化检查通过不等于完成真实 Windows 验收。每个 release 候选版本都应在干净系统或可恢复虚拟机快照中执行以下清单，并记录系统版本、安装包哈希、测试时间和结果。

## 安装、升级与卸载

- [ ] 干净 Windows 10 x64 安装、启动和卸载。
- [ ] 干净 Windows 11 x64 安装、启动和卸载。
- [ ] 未预装 WebView2 时，安装器能正确安装或提示运行时要求。
- [ ] 当前用户安装不要求管理员权限；开始菜单入口可用。
- [ ] 安装路径包含空格时能启动、升级和卸载。
- [ ] 安装路径或 Windows 用户名包含中文时能启动、升级和卸载。
- [ ] 使用更高 `0.1.x` 版本覆盖安装，设置和角色选择仍保留。
- [ ] 覆盖安装后开机启动项指向新版本的有效可执行文件。
- [ ] 卸载后程序、开始菜单和开机启动项已移除。
- [ ] 卸载器退出后，QA 以 500 ms 间隔轮询卸载记录、安装目录、进程、开机启动项和开始菜单，全部清理即通过；最多等待 60 秒，并记录实际耗时。
- [ ] 明确记录卸载后是否保留应用数据目录及 `settings.json`；若保留，重新安装可读取。

## 进程、托盘和生命周期

- [ ] 连续启动两次只保留一个实例，第二次启动会显示并聚焦现有窗口。
- [ ] 单实例锁不会在正常退出、系统重启或崩溃恢复后阻止启动。
- [ ] 托盘“隐藏”后左键托盘图标可以恢复并聚焦窗口。
- [ ] 界面右键菜单“退出”后窗口、托盘图标和进程全部消失。
- [ ] 托盘菜单“退出”后窗口、托盘图标和进程全部消失。
- [ ] 开启和关闭开机启动后，任务管理器/注册项状态与界面一致。
- [ ] 睡眠与唤醒后窗口、动画、托盘和互动正常。
- [ ] Windows 重启后，开启开机启动时只出现一个实例，关闭时不自动启动。

## DPI、显示器和窗口恢复

- [ ] 在 100%、125%、150%、175% 和 200% 缩放下检查显示尺寸、点击区域、拖动阈值和菜单位置。
- [ ] 单显示器环境拖动和位置保存正常。
- [ ] 双显示器相同缩放与相同分辨率环境正常。
- [ ] 双显示器不同缩放和不同分辨率环境正常。
- [ ] 将副显示器设置为主显示器后窗口仍可见。
- [ ] 主副显示器左右、上下和负坐标排列时窗口仍可恢复。
- [ ] 将桌宠放在副显示器后断开显示器，窗口自动回到可见区域。
- [ ] 重新连接显示器后不会抖动、循环移动或越界。
- [ ] 缩放比例运行中变化时锚点和点击区域仍与角色对齐。

## 全屏、交互和动画

- [ ] 全屏游戏、无边框游戏和全屏视频分别验证自动隐藏。
- [ ] 进入或退出全屏时没有频繁闪烁；短暂焦点切换不会反复隐藏和显示。
- [ ] 桌面、任务栏、开始菜单和资源管理器最大化不会被误判为全屏应用。
- [ ] 透明窗口无色块，透明留白区域不会误触，配置的点击区域有效。
- [ ] 单击、双击、悬停、拖动和落地动作不会互相频繁误判。
- [ ] 动画暂停、继续、左右朝向、缩放和透明度操作正常。
- [ ] 单次动作结束后返回配置状态，低优先级动作不会抢占高优先级动作。
- [ ] 长时间运行时动画无闪白、明显抖动或持续累积延迟。

## 角色包与设置容错

- [ ] 新角色只增加目录并运行 `npm run validate:characters` 即可接入。
- [ ] 从设置入口打开“外观中心”后只出现一个窗口；重复点击、快速双击或已有窗口被遮挡时复用并聚焦现有窗口，不创建重复实例。
- [ ] 外观中心同时显示内置角色和 `%LOCALAPPDATA%\dev.deskpet.framework\characters\` 中的本地角色，并以“内置/本地”和“公开/私人”标签明确区分来源与用途。
- [ ] 在内置角色与本地导入角色之间反复切换，缩放、动画、设置持久化和重启后的当前选择均正确。
- [ ] 从包含中文和空格的路径导入合法 `.qipet`，包根、manifest、`frames.json`、PNG 和 SHA-256 相关日志均正常，导入完成后无需修改核心代码。
- [ ] 当前角色导入更高 SemVer 的同 ID 包后，帧、缩略图和图标立即显示新版本；两帧循环可持续往返，不会停在第二帧。
- [ ] 角色预加载期间触发资源重载会收到明确取消结果并恢复所有按钮；主窗口无响应时，切换请求在 110 秒失效，外观中心最迟 120 秒恢复操作，过期请求不得随后生效。
- [ ] 在角色升级的目录切换阶段模拟异常终止后，重新启动会通过事务日志恢复可用版本；不得同时留下可选中的半安装角色。
- [ ] 损坏 ZIP、路径穿越、符号链接/重解析点、不兼容 `schemaVersion`、缺失 `frames.json`、ID 冲突或损坏 PNG 的包被明确拒绝，临时目录和半安装目录不会残留。
- [ ] 当前正在使用的本地角色不能删除；切换到其他角色后才允许删除，删除失败不会让当前角色设置指向不存在的 ID。
- [ ] 覆盖安装升级后，内置角色索引更新，本地角色与当前选择仍保留；同 ID/版本冲突不会被静默覆盖。
- [ ] 外观中心显示 manifest 中的作者、版本、许可和缩略图；公开内置角色缺少 `metadata/source.md` 或 `metadata/license.md` 时构建失败，私人本地包不会被误标为官方或可公开再分发。
- [ ] 角色 manifest 损坏时显示具体错误并回退到 `_placeholder`。
- [ ] 可选动作缺少部分帧时剔除损坏帧，其他帧继续播放。
- [ ] 可选动作全部缺失时回退到 `idle`。
- [ ] 角色 `idle` 全部缺失或损坏时回退到 `_placeholder`。
- [ ] 不连续编号、超大图片、异常 FPS、越界路径和不兼容版本会被校验器拒绝。
- [ ] 设置文件包含截断 JSON、错误类型或越界数值时使用默认值启动。
- [ ] 重启后位置、缩放、透明度、角色、皮肤、置顶和互动设置保留。
- [ ] release 构建没有开发者面板入口；开发诊断构建的模拟工具不会破坏真实设置文件。
- [ ] 卸载后记录 `%LOCALAPPDATA%\dev.deskpet.framework\characters\` 的保留策略：默认保留用户导入角色，重新安装可重新发现；只有用户明确要求清除本地数据时才另行删除，不由 QA 自动清理制造通过结果。

## 性能与稳定性

- [ ] 连续运行至少 8 小时，记录启动后、1 小时、4 小时和 8 小时的内存。
- [ ] 待机状态 CPU 占用稳定，无持续增长或周期性尖峰。
- [ ] 反复切换角色、重载资源和播放动作后图片缓存没有无界增长。
- [ ] 日志不包含凭据、令牌、聊天内容或不必要的绝对本地路径。
- [ ] 异常退出后日志足以定位资源或窗口问题，且下次可以正常启动。

## WebView2 专项

- [ ] 在干净 Windows 10 和 Windows 11 中确认 Evergreen WebView2 Runtime 的初始状态。
- [ ] 已安装 WebView2 时，NSIS 安装不重复下载安装且应用可以启动。
- [ ] 未安装 WebView2且联网时，`downloadBootstrapper` 能完成 Runtime 下载和静默安装。
- [ ] 未安装 WebView2且断网时，安装器给出明确失败，不留下“已安装但无法启动”的伪成功状态。
- [ ] 在代理、TLS 检查或企业软件限制下载时记录安装器退出码和提示。
- [ ] 先安装官方 Evergreen Offline Installer 后，离线安装桌宠成功。
- [ ] 运行 `scripts/check-windows-env.ps1` 能正确报告本机 WebView2 版本或缺失状态。

## 工作区、DPI 和恢复补充

- [ ] 分别验证任务栏位于底部、顶部、左侧和右侧；恢复位置不进入任务栏占用区域。
- [ ] 主屏与副屏使用 100%/125%、125%/150%、150%/200% 等不同 DPI 组合。
- [ ] 副屏位于主屏左侧、上方和左上方，确认负物理坐标保存与恢复正常。
- [ ] 桌宠位于副屏时断开该显示器，确认回到剩余主屏工作区。
- [ ] 睡眠、唤醒和远程桌面连接/断开后，窗口仍可见且不会持续跳动。
- [ ] 极端窗口尺寸和系统缩放变化后，至少保留可拖动的可见区域。

Rust 单元测试只验证坐标和 DPI 的纯逻辑，不代表上述真实显示器硬件测试已经通过。

## 验证状态分类

- 自动验证：类型检查、前端单元测试、角色包校验、Rust 单元测试、生产构建和 NSIS 构建。
- 本机人工验证：Release 启动、占位角色显示、单实例、普通 Production 不显示开发工具、右键菜单可达和正常退出无残留进程。
- 尚需干净 Windows：WebView2 缺失、安装、覆盖升级、卸载、中文用户目录和受限网络。
- 尚需真实硬件：多显示器、混合 DPI、任务栏位置、显示器热插拔、睡眠/唤醒和远程桌面。
- 尚需长时间运行：1 小时和 8 小时 CPU、内存、句柄、线程及日志增长。执行方法见 `docs/PERFORMANCE_TEST.md`。

统一执行入口：

```powershell
.\scripts\windows\run-qa-suite.ps1 -Mode Safe
.\scripts\windows\run-qa-suite.ps1 -Mode CurrentMachine -WhatIf
```

真实显示器填写模板见 `MANUAL_DISPLAY_QA.md`；Sandbox/VM、WebView2、签名和 SmartScreen 分别见对应专项文档。`CleanEnvironment` 模式要求测试环境内显式设置 `DESK_PET_QA_CLEAN_ENVIRONMENT=1`，不得通过主机名猜测环境性质。

公开测试版总清单见 `PUBLIC_BETA_CHECKLIST.md`。统一环境入口和 Gate 审核：

```powershell
.\scripts\windows\run-public-beta-qa.ps1 -Mode Safe -WhatIf
.\scripts\windows\run-public-beta-qa.ps1 -Mode Sandbox
.\scripts\windows\run-public-beta-qa.ps1 -Mode CleanWindows10
.\scripts\windows\run-public-beta-qa.ps1 -Mode CleanWindows11
.\scripts\windows\run-public-beta-qa.ps1 -Mode PublicBetaAudit
```

`Sandbox` 和 VM 模式默认只检测并记录环境，不会启用 Windows 功能、启动 VM、创建快照或安装软件。实际升级使用两个不同版本的安装包和显式可丢弃环境标记；同版本重复安装不算升级。
