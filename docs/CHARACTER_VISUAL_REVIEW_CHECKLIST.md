## 0. 开始审核前

- [ ] 用户已确认角色类型、画风、比例、写实程度、配色、性格与标志性特征。
- [ ] 用户已确认首批动作、每个动作的目标帧数和动画形式。
- [ ] 如使用真人、宠物或其他参考图，已确认来源和允许的改编范围。
- [ ] 已确认当前白猫应用图标是否允许作为角色参考；未明确确认时，角色没有从该图标推导。
- [ ] 当前交付仍采用 `schemaVersion: 1` 的 PNG 序列帧正式路径。
- [ ] 角色通过 `npm run validate:characters`，且没有通过跳过或降低校验规则制造结果。
- [ ] 已准备正常速度、`0.5x`、`0.25x`、逐帧、锚点、hitbox 和画布边界检查方式。

## 1. 技术基础门槛

以下项目通过，只代表正式素材可以进入视觉验收，不代表美术完成：

- [ ] 占位资源和候选素材连续播放时没有空白帧、白闪或损坏图片图标。
- [ ] 动画速度不随 60 Hz、120 Hz、144 Hz 等显示器刷新率改变。
- [ ] 暂停、恢复、窗口隐藏再显示后没有快速补帧。
- [ ] 状态切换时不会显示尚未加载或尚未解码的图片。
- [ ] 拖动结束后不会误触发点击或双击。
- [ ] 低优先级状态不会打断高优先级或不可中断状态。
- [ ] 随机行为不会连续重复同一动作，并保留合理安静时段。
- [ ] 多次角色包重载后内存不持续线性上涨。
- [ ] 开发者显示的锚点、hitbox 和画布边界与实际素材一致。
- [ ] Production 构建中不出现开发者工具入口或调试覆盖层。
- [ ] 新角色仅通过资源包接入，不需要修改核心业务代码。
- [ ] 现有 `schemaVersion: 1` 旧角色包仍完全兼容。

技术基础结论：通过 / 不通过

## 2. 文件与画布

- [ ] 所有帧是可读取的 RGBA 透明 PNG。
- [ ] 所有帧画布尺寸与 manifest 的 `frameSize` 一致。
- [ ] 文件名使用固定四位、从 1 连续递增的编号。
- [ ] 没有逐帧自动裁切造成的画布或构图变化。
- [ ] 角色实际尺寸和四周透明留白在动作间保持统一。
- [ ] 预览图与实际角色包内容、比例和最新版本一致。

## 3. 造型一致性

- [ ] 不同动作中的角色整体比例一致。
- [ ] 摄像机视角、透视和观察高度一致。
- [ ] 角色服装、配色、材质和细节一致。
- [ ] 没有随机增减、替换或漂移的饰品。
- [ ] 手指、耳朵、角、翅膀、尾巴等数量和结构稳定。
- [ ] 发型、脸型、五官、花纹和标志性特征没有漂移。
- [ ] 次级运动没有改变头发、衣物、耳朵或尾巴的固定点和基本长度。
- [ ] 快速播放和逐帧查看时，角色仍能被识别为同一设计。

## 4. 锚点、接地与运动

- [ ] 脚底或身体锚点在所有帧中一致。
- [ ] 动作切换没有非设计意图的横向或纵向跳位。
- [ ] 站立和行走时角色与桌面具有稳定接地感。
- [ ] 行走步频与实际移动速度协调，没有明显滑步。
- [ ] 起步、减速、停止和转向有合理重心变化。
- [ ] 角色接近屏幕边缘时动作与位移保持协调。
- [ ] 100%、125%、150%、175% 和 200% 缩放下运动观感基本一致。
- [ ] 多显示器和负坐标场景下，角色没有改变比例或突然跳到无关位置。

## 5. 动作结构与表演

对每个 `idle`、行走、睡眠、悬停、点击、双击、拖动及其他候选动作逐项检查：

- [ ] 动作意图清晰，不依靠声音掩盖视觉反馈不足。
- [ ] 需要时包含前摇、主体和收尾，而不是瞬间进入极端姿势。
- [ ] 动作力度、节奏和角色性格一致。
- [ ] 动作结束姿势能够自然进入 manifest 指定的回退状态。
- [ ] 循环点没有停顿、抽动、跳位、亮度变化或轮廓闪烁。
- [ ] 呼吸起伏自然、幅度克制，没有机械等速缩放。
- [ ] 眨眼节奏自然，没有与每次呼吸机械同步。
- [ ] 表情变化连续，没有五官漂移或瞬间换脸。
- [ ] 头发、服装、耳朵、尾巴等次级运动与主体动作协调。
- [ ] 没有用随机缩放、随机位置或随机细节变化伪造生命感。

## 6. 动作衔接

- [ ] `idle` 与左右行走可以双向自然衔接。
- [ ] 起步和停止没有突然达到最高速度或瞬间定住。
- [ ] 睡眠前有合理进入过程，唤醒后状态一致。
- [ ] `hover`、`click`、`double_click` 完成后自然回到合理状态。
- [ ] 拖动进入时锚点稳定，释放后有克制的恢复过程。
- [ ] 连续交互不会形成无限队列或导致角色在状态间震荡。
- [ ] 高优先级交互打断低优先级动作时，画面仍连续且原因合理。
- [ ] 方向改变不会连续快速左右抖动。

## 7. 左右方向

- [ ] 左右行走素材的角色尺寸、锚点、步频和光影一致。
- [ ] 文字、徽记、发饰、服装开口等非对称细节方向正确。
- [ ] 不允许镜像的素材已提供独立左右方向帧。
- [ ] 允许镜像的素材已经人工确认，不会产生文字反转、结构错误或光影矛盾。
- [ ] 左右方向切换时没有角色身份特征变化。

## 8. 光影、透明边缘与阴影

- [ ] 所有动作的主光源方向和明暗逻辑一致。
- [ ] 浅色和深色背景上均没有白边、黑边、色边或脏像素。
- [ ] 半透明区域没有闪烁、破洞或逐帧颜色污染。
- [ ] 图片内没有棋盘格、纯色底或其他预览背景。
- [ ] 没有把不适合所有地面、方向或动作的阴影烘焙进角色帧。
- [ ] 若使用独立接触阴影，其位置跟随锚点，透明窗口没有方形底色。

## 9. hitbox 与实际显示

- [ ] 可点击区域覆盖预期角色主体，不包含过多透明留白。
- [ ] 细长耳朵、尾巴、配饰等区域的点击策略符合交互设计。
- [ ] 调试 hitbox 与实际输入命中结果一致。
- [ ] 默认缩放下角色清晰且不过度占用桌面。
- [ ] 放大和缩小时没有明显采样边缘、抖动或动作比例改变。
- [ ] 窗口边缘没有裁掉角色、次级运动或交互反馈。

## 10. 来源、授权与版权

- [ ] `metadata/source.md` 记录了原创、委托、用户提供和所有第三方来源。
- [ ] `metadata/license.md` 记录版权所有者、修改、商业使用、发布、再分发和署名条件。
- [ ] 授权证据可追溯，公开文档中没有本机绝对路径或敏感个人信息。
- [ ] 没有未经授权的现有版权角色或高度可识别的仿冒设计。
- [ ] 没有未经授权的商标、品牌标识、字体、纹理、模型或素材包。
- [ ] 真人肖像、宠物照片和私人图片已取得适用于角色化和发布的许可。
- [ ] 生成式工具辅助内容的输入参考和输出用途均符合计划中的发行方式。
- [ ] 已由人工检查生成内容是否与现有角色过度近似或包含异常版权元素。
- [ ] “网络可见”“用户上传”“AI 生成”没有被当作授权证明。
- [ ] 如任何来源或权利范围不明确，候选素材已被阻止进入正式发布包。

## 11. 最终结论

只有下列结论全部成立，才能标记为“正式美术验收通过”：

- [ ] 技术基础门槛通过。
- [ ] 文件、造型、动作、衔接、方向、边缘和实际显示的人工审核通过。
- [ ] 用户或指定美术负责人确认角色方案和表现符合预期。
- [ ] 来源、版权和发布授权完整。
- [ ] 所有不通过项已修复并复验，没有用“不适用”规避必要检查。

如果只通过技术检查，应记录为“技术可接入，待正式美术验收”。如果当前仍使用中性几何占位素材，应记录为“动画基础验证”，不得写成“正式角色已完成”“已达到生动真实”或类似结论。
