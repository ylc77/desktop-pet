# 已知问题

## 严重/高优先级

### 生产自动更新尚未配置和端到端验证

- 状态：`BLOCKED`，阻塞公开测试版。
- 当前基础：Tauri 2 Updater/Process、更新状态与用户入口已经集成；普通构建保持 `NOT_CONFIGURED` 且不联网。发布工具已实现 artifact + `.sig` + 外部公钥的真实密码学验签，并校验元数据的精确版本、URL 文件名和实际 `size`。
- 缺失：生产 updater 私钥与至少两份备份、正式公钥、用户确认的 HTTPS endpoint、最终 `.sig`/`latest.json`，以及两个真实版本的下载、安装、重启和数据保留验证。
- 注意：工具验签能力、临时测试密钥或普通 Release 构建通过都不能作为生产更新已可用的证据。不得填写虚假 URL、禁用签名验证或提前上传安装包。

### 当前候选的完整 Windows 生命周期证据缺失

- 状态：开放，阻塞公开测试版。
- 历史背景：改名前的 `Desk Pet Framework 0.1.0` 曾出现卸载器返回后旧安装目录、注册表记录和开始菜单项仍存在的报告；该历史结果不能自动判定当前候选通过或失败。
- 当前要求：用本候选重新执行真实安装、启动、单实例、正常退出、开机启动和卸载，并以最多 60 秒的有界轮询记录清理时间。
- 注意：不得手工删除注册表、目录或未知文件来制造通过结果。

## 公开测试风险

- 安装包当前 `NotSigned`，存在 SmartScreen 和未知发布者风险。
- Updater 签名只保护更新包完整性，不能替代 Windows 代码签名。
- 干净 Windows 10/11、WebView2 缺失联网/断网、真实升级尚未执行。
- 真实多显示器、混合 DPI、睡眠、唤醒、重启和远程桌面尚未执行。
- 尚无完整 8 小时稳定性结果。
- 当前素材是中性占位角色。
- 品牌改名后的安装、开始菜单、应用与功能、托盘和卸载全流程需要重新执行真实 QA。

## 可延期能力

ARM64、完全离线 WebView2、Microsoft Store、macOS 和 Linux 不在当前版本范围。自动更新基础已经进入范围，但生产配置与端到端测试尚未完成，因此不能列为“延期后仍可公开”的项目。
