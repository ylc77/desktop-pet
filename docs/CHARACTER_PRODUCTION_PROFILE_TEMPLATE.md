# 角色制作规格模板

每次开始制作一个新角色时，由 Codex 根据用户提供的设定和参考资料复制、填写并提出首版建议，用户确认后再制作素材。该文档是制作记录，不是 `manifest.json` 的新字段，也不改变 `schemaVersion: 1`。

不要从其他角色机械复制帧数和 FPS。动作预算必须结合本角色画风、目标显示尺寸、动作复杂度、包体和运行时性能重新计算。

## 1. 基本设定

| 项目 | 本角色决定 |
| --- | --- |
| 角色 ID / 显示名称 / 版本 | 待填写 |
| 角色类型 | 人物 / 动物 / 真人角色化 / 宠物 / 游戏题材 / 机器人 / 幻想生物 / 其他 |
| 画风与写实程度 | 待填写 |
| 性格、主色与识别特征 | 待填写 |
| 服装、发型、配饰或品种特征 | 待填写；不同完整服装首版使用独立角色包和独立 ID |
| 用途标签 | 公开内置 / 私人本地 |
| 目标显示高度 | 待填写逻辑像素，并注明 100% 与高 DPI 下的检查尺寸 |
| 固定画布与角色占用比例 | 待填写宽高、透明留白范围 |
| 锚点与 hitbox | 待填写归一化值及选择理由 |
| 方向策略 | 独立左右素材 / 允许镜像；列出不能镜像的文字、徽记和不对称细节 |
| 光源与阴影 | 待填写；说明是否使用程序接触阴影 |

## 2. 动作 production profile

Codex 应先列关键姿势，再选择能自然呈现这些姿势的最低有效帧数和 FPS。下表每个动作单独填写，不允许用一个全局数字覆盖全部动作。

| 动作名 | 触发与表现目的 | 复杂度 | 前摇关键姿势 | 主体关键姿势 | 收尾关键姿势 | 帧数 | FPS | 循环/时长 | returnTo | 方向/移动 | 选择理由 |
| --- | --- | --- | --- | --- | --- | ---: | ---: | --- | --- | --- | --- |
| `idle` | 待填写 | 低/中/高 | 待填写 | 待填写 | 待填写 | 待定 | 待定 | 待定 | 待定 | 待定 | 结合目标尺寸与细节说明 |
| 其他动作 | 待填写 | 低/中/高 | 待填写 | 待填写 | 待填写 | 待定 | 待定 | 待定 | 待定 | 待定 | 待填写 |

至少检查实际需要的 `idle`、移动、悬停、单击、双击、拖动、落地、睡眠/唤醒；未纳入首版的动作写明原因，不用为了凑清单制作低质量空动作。

## 3. 质量、体积与性能预算

| 指标 | 预算与结果 |
| --- | --- |
| 动作总帧数 | 待填写 |
| 最大单帧 PNG / 总压缩包体 | 待填写目标与实测值 |
| 单帧 RGBA 估算 | `画布宽 × 画布高 × 4 bytes` |
| 预计同时保留帧数 | 待填写；结合播放器约 64 MiB 解码引用预算 |
| 预计解码引用占用 | `单帧 RGBA 估算 × 预计保留帧数` |
| 目标测试设备 | 待填写 Windows 10/11、内存、GPU、DPI |
| 质量/性能取舍 | 说明哪些动作保留更多关键帧，哪些通过缩小画布、优化透明留白或降低有效 FPS 控制开销 |

验收必须在目标尺寸、`0.25x`、`0.5x` 和 `1x` 播放速度下检查，确认关键姿势清晰、循环无跳帧、移动无滑步，且切换角色和重载后内存不会无界增长。

## 4. 来源与授权

| 项目 | 记录 |
| --- | --- |
| 原创、委托、用户提供或第三方来源 | 待填写；不记录敏感本机绝对路径 |
| 游戏角色、商标或品牌元素 | 待填写权利人和允许范围；自行重绘不自动取得公开发行权 |
| 真人或宠物参考 | 待填写照片权利、本人/所有者同意和角色化使用范围 |
| 生成式工具、模型、纹理、字体、笔刷 | 待填写工具、许可与可再分发范围 |
| `metadata/source.md` | 待填写 |
| `metadata/license.md` | 待填写发布、修改、商业使用、再分发和署名条件 |

私人本地包必须标为“私人/本地”，不得自动上传或纳入安装包。准备公开分发时，即使素材此前只在本机使用，也必须重新按公开角色标准审核授权。

## 5. 交付确认

- [ ] 用户已确认角色设定、画风、目标尺寸与用途标签。
- [ ] 用户已确认 Codex 提出的逐动作帧数、FPS、循环、前摇和收尾方案。
- [ ] `manifest.json` 与实际动作、帧数、方向和返回状态一致。
- [ ] `preview.png`、`icon.png`、`metadata/source.md` 和 `metadata/license.md` 完整。
- [ ] `npm run validate:characters` 通过并生成 `frames.json`。
- [ ] 人工视觉清单通过，性能预算与实测结果已记录。
- [ ] 需要本地导入时，`.qipet` 根结构、大小和 SHA-256 已复核。
