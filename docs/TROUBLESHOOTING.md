# 七酱桌宠故障排查

## 无法启动

- 确认 Windows 10/11 x64 和 Microsoft Edge WebView2 Runtime 可用。
- 检查 `%LOCALAPPDATA%\dev.deskpet.framework\logs\` 中的最近日志。
- 不要安装 Node.js 或 Rust；最终用户运行不需要开发工具。

## 窗口不可见

- 再次启动七酱桌宠，以触发单实例窗口恢复。
- 通过系统托盘中的七酱桌宠图标尝试恢复。
- 显示器刚断开或 DPI 改变时，等待窗口恢复到现有工作区。

## 设置损坏

应用会回退到默认设置，并把损坏文件改名为 `settings.corrupt-<时间戳>.json`。不要公开上传包含本机路径或私人内容的设置和日志。

也可以从“关于七酱桌宠”执行“恢复默认设置”。该操作需要二次确认，只重置设置，不删除角色包、日志或应用；完成后窗口会恢复到可见区域。

## 更新服务尚未配置

当前 `0.1.0` 的生产 updater 公钥和 HTTPS endpoint 尚未配置。看到“更新服务尚未配置”是预期状态，不代表网络或系统故障；应用不会在后台持续重试，也不能通过修改本地文件绕过签名验证。

正式配置后仍检查失败时：

- 确认系统时间、HTTPS 访问、代理和证书链正常。
- 手动重试一次，并记录用户界面显示的失败类别。
- 不要关闭 TLS、改用 HTTP、禁用签名验证或用 `Invoke-Expression` 运行下载内容。
- 404、超时、非法 JSON、空/错误签名、下载中断和安装失败需要分别记录，不能统一标记为“无网络”。

签名验证失败时继续使用当前已安装版本，不运行下载文件；把脱敏诊断交给维护者。Windows“未知发布者”则属于当前 Authenticode `NotSigned` 风险，不等于 Tauri Updater 签名校验失败。

若安装命令异常返回且受控重启兜底失败，界面会记录 `restartFailed`。此时“重试”只再次尝试启动新进程，不会重复运行安装程序。若重启后实际版本仍不是 pending 目标，应用保留失败目标作为诊断证据，并只提示一次；不要手工删除 pending 数据来伪造更新成功。

## 安装失败

- 缺少 WebView2 且断网时，当前小体积安装包不能完成完全离线安装。
- 记录安装器退出码、Windows 版本和安装包 SHA-256。
- 不要关闭 Defender、添加排除项或绕过 SmartScreen。

## 卸载后仍有残留

先确认没有 `desktop_pet.exe` 或卸载器进程。保留现场并记录卸载注册表、安装目录和开始菜单状态，不要直接删除注册表来伪造通过结果。已知问题请参阅 `KNOWN_ISSUES.md`。

## 导出诊断

从“关于七酱桌宠”主动导出脱敏诊断，并在交给维护者前自行检查。应用不会自动上传。诊断包不应包含用户名、完整本地路径、网络共享路径、私钥、密码、令牌、私人角色素材或带敏感 query 的 URL；发现这些内容时不要发送，并保留最小复现步骤供修复脱敏逻辑。

项目当前没有公开客服或自动反馈渠道。请把最小复现步骤和脱敏后的 QA 报告交给项目维护者。
