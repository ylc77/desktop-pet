# 0.1.2-beta.3 公开测试版候选说明（草案）

当前发布状态：`NOT_READY`。

这是一份修复候选说明草案。`0.1.2-beta.3` 已完成未签名的本地 Release 构建，但尚未在干净提交上生成生产 Updater 签名或发布。已验证的 `0.1.2-beta.2` Draft 与五个远端资产将保持不变；本候选必须在干净提交上重新构建、重新生成 Updater 签名与元数据，并在独立 Draft 中验证。

七酱桌宠是离线优先、角色资源包驱动的 Windows 桌宠应用。当前包含透明置顶窗口、PNG 动画状态机、拖动和点击交互、设置持久化、托盘、单实例、开机启动、全屏隐藏、日志和资源校验。当前画面仍为带 `DEV` 标识的中性几何占位角色，不是正式桌宠外观。

## 本候选修复

- 依据当前显示器工作区与缩放比例限制设置和外观中心窗口，避免 200% 缩放或小工作区下关键操作不可达。
- 提升浅色、深色及跟随系统主题的文字和控件边界对比度，并增加可计算的对比度回归测试。
- 让系统“减少动画”动态控制桌宠序列帧和移动，而不只停用 CSS 过渡。
- 合并连续设置更新，确保拖动滑块时最终值不会因前一个保存请求仍在进行而丢失。
- 让设置变更、角色切换、启动恢复与安全退出共用顺序化事务队列；失败时回滚原生状态和持久化数据，退出前保存最终窗口位置。
- 允许删除当前本机角色，并先安全回退到有效的内置占位角色；删除成功、失败和取消后均恢复键盘焦点。
- 收紧 CSP 与 `.qipet` 允许的资源类型，拒绝脚本、可执行文件和 HTML 等无关内容。
- 补全 `file:///` 等本机路径的诊断脱敏测试。
- 让发布校验按当前候选版本选择 Updater 证据；保留的历史候选不会污染未签名基础构建，当前版本的残缺签名证据仍会严格失败。
- 增加专有软件许可、第三方组件声明和白猫图标/中性占位素材的公开测试分发授权记录。

角色包协议仍为 `schemaVersion: 1`，identifier 仍为 `dev.deskpet.framework`，白猫应用图标与中性占位素材未改变。本候选不包含正式桌宠角色，也不包含 AI、账号、支付、云数据库、遥测、广告或在线诊断上传。

生产签名构建使用 HTTPS GitHub Releases endpoint 与既有生产公钥；普通构建的 Updater 配置保持离线且不会发起更新请求。发现更新后，必须由用户点击“立即更新”才会下载、验签并交接 NSIS 安装，不会静默下载或安装。

## 当前限制

- 仅面向 Windows 10/11 x64；干净 Windows 10/11、WebView2 缺失、真实混合 DPI 双屏、睡眠/唤醒和 8 小时测试尚未完成。
- Windows Authenticode 尚未配置，安装包状态预计仍为 `NotSigned`，可能触发 SmartScreen 或显示未知发布者。Tauri Updater 签名不能替代 Windows 代码签名。
- `0.1.2-beta.3` 已完成未签名的本地 Release 构建，但尚未完成干净提交构建、生产 Updater 签名、远端回下载、真实安装/卸载及端到端升级验证；旧候选结果不能替代本候选证据。
- WebView2 缺失时安装器需要联网下载运行时，尚不支持完全离线安装。

项目当前没有公开客服或自动反馈服务。诊断包只由用户主动导出到本地，不会自动上传；测试人员应先自行检查内容，再将必要的 QA 结果、安装包 SHA-256、Windows 版本和复现步骤交给项目维护者。
