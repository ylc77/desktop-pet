# 关于、诊断与恢复默认设置

公开测试基础在主窗口右键菜单、托盘菜单和设置中提供“检查更新”与“关于七酱桌宠”入口。更新渠道为 `beta`；普通构建未配置时明确显示该状态，生产签名构建则显示已配置的 endpoint 域名与公钥指纹。

## 关于页面

关于页面面向普通用户显示：

- 七酱桌宠和当前应用版本
- 白猫应用图标
- 当前更新渠道与最近一次检查时间
- 启动自动检查开关；修改后与设置页使用同一持久化配置
- 检查更新、一次点击“立即更新”，以及 `postponed`、`skipped`、`restartFailed` 等更新状态
- 打开日志目录
- 导出诊断信息
- 隐私说明和已知问题入口

identifier 不需要作为普通信息突出显示；内部开发更新入口不能出现在 Production。

已知问题文案必须根据运行时 updater 配置显示：未配置构建才显示“生产更新密钥和公网更新地址尚未配置”；配置完整的构建改为提示“自动更新已配置，但真实两版本升级仍待验证”，不得在已配置版本中继续写“未配置”。

## 导出诊断

诊断导出必须由用户主动发起，并生成 `qijiang-diagnostics-<时间戳>.zip` 到本机下载目录；下载目录不可用时回退到应用数据目录。应用不会自动上传、发送或同步诊断内容，界面会显示实际文件名与保存位置类别。

压缩包只包含白名单生成的 `diagnostics.json`、`recent-frontend-logs.json`、`recent-native-logs.txt` 和本地说明。原生日志最多读取最近 256 KiB，并在写入 ZIP 前逐行脱敏；不会把整个设置文件、角色目录或任意磁盘文件打包。

允许包含：

- 应用版本、Windows 版本和架构
- Tauri 与 WebView2 版本（能够安全获取时）
- 当前角色包 ID、版本和 `schemaVersion`
- 不含自由文本与绝对路径的关键设置摘要
- 最近的脱敏日志
- updater 状态和最近失败类别
- updater endpoint 的域名或不可逆摘要，不含完整路径、query 或凭据
- 当前安装包哈希（能够可靠获得时）

必须排除或脱敏：

- 用户真实姓名、用户名和完整用户目录
- 私钥、公钥以外的密钥材料、密码、令牌和授权头
- 完整本地路径、UNC 网络共享路径和带敏感 query 的 URL
- 原始设置文件、私人角色素材、参考照片和个人内容
- 任何聊天内容（当前应用本身不包含聊天功能）

导出后由用户自行审阅，再决定是否交给维护者。当前没有在线反馈或自动诊断上传服务。

## 日志目录

“打开日志目录”只打开 `%LOCALAPPDATA%\dev.deskpet.framework\logs\`，不上传文件。日志轮转策略仍为单文件最多 1 MiB、最多 5 个轮转文件。更新错误记录分类、目标版本和脱敏域名，不记录签名私钥、密码或完整敏感 URL。

## 恢复默认设置

恢复默认设置必须经过二次确认，并明确说明只会重置窗口位置、缩放、互动、置顶、全屏隐藏和更新偏好；开机启动、透明度、音量、当前角色和更新事务确认状态保持不变。操作完成后：

- 不删除内置或本地角色包。
- 不删除日志、应用或安装记录。
- 不改变 identifier 或角色协议。
- 让桌宠窗口恢复到当前显示器可见区域。
- 继续使用稳定 identifier 派生的数据目录。

## 完全清除本地数据

本轮不提供“一键清除全部数据”按钮。需要完全清理时，先正常退出并卸载应用，再由用户明确删除以下应用专属目录：

- `%APPDATA%\dev.deskpet.framework`
- `%LOCALAPPDATA%\dev.deskpet.framework`

不要删除它们的父目录，也不要让 QA 脚本自动删除本地数据来制造通过结果。
