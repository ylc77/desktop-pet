# 项目原则

1. 当前项目是通用 Windows 桌宠开发框架。
2. 技术栈使用 Tauri 2、React、TypeScript 和 Rust。
3. 当前阶段不加入AI聊天、账号、支付、云数据库等功能。
4. 正式桌宠外观未确定前，只使用占位素材。
5. 需要制作正式外观时，必须先询问用户期望的角色和画风。
6. 角色素材必须通过资源包加载，不得硬编码进业务逻辑。
7. 每次完成修改后运行：
   - 前端类型检查
   - Rust cargo check
   - 现有测试
   - release构建检查
8. 不得为了通过构建删除现有功能或跳过错误。
9. 优先保证Windows 10和Windows 11兼容。
